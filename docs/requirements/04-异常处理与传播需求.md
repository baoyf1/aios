## 一、异常传播机制

### 1.1 设计原则

```
┌─────────────────────────────────────────────────────────────────────┐
│                         异常传播原则                                  │
│                                                                      │
│  核心思想:                                                           │
│  - 每层只处理本层能处理的异常                                         │
│  - 无法处理的异常转换为该层的异常类型向上抛出                         │
│  - 最终在应用层转换为用户友好的错误消息                               │
│                                                                      │
│  类比Linux:                                                          │
│  - 内核处理硬件异常，转换为信号通知进程                               │
│  - 进程处理信号，执行信号处理程序                                     │
│  - 最终进程崩溃生成core dump或错误日志                               │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 传播流程图

```
                        ▲ 异常向上传播 (类似异常穿透)
                        │
┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  Layer 1 (硬件)                                                      │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  发现异常:                                                    │  │
│  │  - LLM API调用失败 (网络超时、API错误)                       │  │
│  │  - 文件不存在                                                 │  │
│  │  - 向量数据库连接失败                                         │  │
│  │                                                              │  │
│  │  处理: 记录详细错误日志                                       │  │
│  │  抛出: HwException                                           │  │
│  └───────────────────────────────────────────────────────────────┘  │
│        ▲                                                               │
│        │ 捕获                                                      │
│  Layer 2 (内核)                                                      │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  捕获 HwException                                             │  │
│  │  处理:                                                        │  │
│  │  - 记录审计日志 (agent_id, action, error_detail)             │  │
│  │  - 增加错误计数                                               │  │
│  │  - 可能触发重试逻辑                                           │  │
│  │  抛出: KernelException (转换)                                 │  │
│  └───────────────────────────────────────────────────────────────┘  │
│        ▲                                                               │
│        │ 捕获                                                      │
│  Layer 3 (系统调用)                                                  │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  捕获 KernelException                                         │  │
│  │  处理:                                                        │  │
│  │  - 检查权限 (PathGuard + CapChecker)                         │  │
│  │  - 转换错误码为统一格式                                       │  │
│  │  抛出: SysCallException (转换)                               │  │
│  └───────────────────────────────────────────────────────────────┘  │
│        ▲                                                               │
│        │ 捕获                                                      │
│  Layer 4 (中间件)                                                    │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  捕获 SysCallException                                       │  │
│  │  处理:                                                        │  │
│  │  - 序列化错误信息                                             │  │
│  │  - 添加中间件上下文                                           │  │
│  │  抛出: MiddlewareException (转换)                             │  │
│  └───────────────────────────────────────────────────────────────┘  │
│        ▲                                                               │
│        │ 捕获                                                      │
│  Layer 5 (应用层)                                                    │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │  捕获 MiddlewareException                                    │  │
│  │  处理:                                                        │  │
│  │  - 转换为用户友好的错误消息                                   │  │
│  │  - 决定是否需要重试                                           │  │
│  │  - 记录到记忆系统                                             │  │
│  │  结果: 返回给用户                                             │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 二、异常类型定义

### 2.1 异常基类

```cpp
// ==================== aios_exception.h ====================
#pragma once

#include <exception>
#include <string>
#include <cstdint>

namespace aios {

// 异常基类
class AIOSException : public std::exception {
public:
    AIOSException(int code, const std::string& msg) 
        : code_(code), message_(msg), timestamp_(get_current_timestamp_ms()) {}
    
    AIOSException(int code, const std::string& msg, const std::string& details)
        : code_(code), message_(msg), details_(details), 
          timestamp_(get_current_timestamp_ms()) {}
    
    virtual ~AIOSException() noexcept = default;
    
    // 获取异常信息
    int code() const { return code_; }
    const char* what() const noexcept override { return message_.c_str(); }
    const std::string& message() const { return message_; }
    const std::string& details() const { return details_; }
    uint64_t timestamp() const { return timestamp_; }
    
    // 转换为JSON
    virtual std::string to_json() const {
        return std::string("{\"code\":") + std::to_string(code_) +
               ",\"message\":\"" + message_ + "\"" +
               ",\"timestamp\":" + std::to_string(timestamp_) + "}";
    }
    
protected:
    int code_;                      // 错误码
    std::string message_;           // 错误消息
    std::string details_;           // 详细信息
    uint64_t timestamp_;           // 时间戳
};

} // namespace aios
```

## 三、异常处理策略

### 3.1 各层处理策略表

| 层级 | 捕获异常 | 处理方式 | 重新抛出 | 记录日志 |
|-----|---------|---------|---------|---------|
| Layer 1 | HwException | 记录错误详情 | 转换为KernelException | 是 |
| Layer 2 | KernelException | 调度器重试/切换Agent | 转换为SysCallException | 是 |
| Layer 3 | SysCallException | 权限检查/资源限制 | 转换为MiddlewareException | 是 |
| Layer 4 | MiddlewareException | 格式化错误信息 | 转换为AppException | 是 |
| Layer 5 | AppException | 用户友好提示 | 不再抛出 | 可选 |

### 3.2 异常转换器

```cpp
// ==================== exception_converter.h ====================
#pragma once

#include "hw_exception.h"
#include "kernel_exception.h"
#include "syscall_exception.h"
#include "middleware_exception.h"
#include "app_exception.h"

namespace aios {

// 异常转换器
class ExceptionConverter {
public:
    // HwException → KernelException
    static KernelException to_kernel(const HwException& e, pid_t involved_pid = PID_INVALID) {
        std::string msg = "Hardware layer error propagated: " + std::string(e.what());
        return KernelException(
            KernelException::KernelErrorCode::IPC_ERROR,
            msg,
            e.details(),
            involved_pid
        );
    }
    
    // KernelException → SysCallException
    static SysCallException to_syscall(const KernelException& e, 
                                       pid_t caller_pid = PID_INVALID,
                                       uint32_t syscall_id = 0) {
        std::string msg = "Kernel layer error propagated: " + std::string(e.what());
        
        // 根据内核错误码转换
        SysCallException::SysCallErrorCode syscall_code;
        switch (static_cast<KernelException::KernelErrorCode>(e.code())) {
            case KernelException::KernelErrorCode::AGENT_NOT_FOUND:
                syscall_code = SysCallException::SysCallErrorCode::DRIVER_NOT_FOUND;
                break;
            default:
                syscall_code = SysCallException::SysCallErrorCode::DRIVER_ERROR;
        }
        
        return SysCallException(syscall_code, msg, e.details(), caller_pid, syscall_id);
    }
    
    // SysCallException → MiddlewareException
    static MiddlewareException to_middleware(const SysCallException& e,
                                             const std::string& session_id = "") {
        std::string msg = "System call layer error propagated: " + std::string(e.what());
        
        MiddlewareException::MiddlewareErrorCode mw_code;
        switch (static_cast<SysCallException::SysCallErrorCode>(e.code())) {
            case SysCallException::SysCallErrorCode::SERIALIZE_ERROR:
            case SysCallException::SysCallErrorCode::DESERIALIZE_ERROR:
                mw_code = MiddlewareException::MiddlewareErrorCode::SERIALIZE_ERROR;
                break;
            default:
                mw_code = MiddlewareException::MiddlewareErrorCode::INVALID_FORMAT;
        }
        
        return MiddlewareException(mw_code, msg, e.details(), session_id);
    }
    
    // MiddlewareException → AppException
    static AppException to_app(const MiddlewareException& e,
                                 const std::string& context = "") {
        std::string msg = "Middleware layer error propagated: " + std::string(e.what());
        
        AppException::AppErrorCode app_code;
        switch (static_cast<MiddlewareException::MiddlewareErrorCode>(e.code())) {
            case MiddlewareException::MiddlewareErrorCode::SESSION_EXPIRED:
                app_code = AppException::AppErrorCode::TASK_FAILED;
                break;
            default:
                app_code = AppException::AppErrorCode::TASK_FAILED;
        }
        
        return AppException(app_code, msg, e.details());
    }
};

} // namespace aios
```

### 3.3 各层处理代码示例

```cpp
// ==================== layer2_handler.cpp ====================
class KernelLayer {
private:
    AuditLog& audit_log_;
    Scheduler& scheduler_;
    
public:
    Result execute(const KernelRequest& request) {
        try {
            // 调用下层 (Layer 1)
            return layer1_execute(request);
        } catch (const HwException& e) {
            // 1. 记录审计日志
            audit_log_.log_error(request.trace_id, request.from_pid, 
                               "HwException", e.to_json());
            
            // 2. 增加错误计数
            error_counter_.increment(e.code());
            
            // 3. 判断是否可重试
            if (is_retryable(e)) {
                // 重试逻辑
                return retry(request);
            }
            
            // 4. 转换为KernelException并重新抛出
            throw ExceptionConverter::to_kernel(e, request.from_pid);
            
        } catch (const KernelException& e) {
            // 已经是KernelException，记录并重新抛出
            audit_log_.log_error(request.trace_id, request.from_pid,
                               "KernelException", e.to_json());
            throw;
        }
    }
    
    bool is_retryable(const HwException& e) {
        // 超时和临时错误可重试
        switch (static_cast<HwException::HwErrorCode>(e.code())) {
            case HwException::HwErrorCode::LLM_TIMEOUT:
            case HwException::HwErrorCode::HTTP_TIMEOUT:
            case HwException::HwErrorCode::IPC_TIMEOUT:
                return true;
            default:
                return false;
        }
    }
};
```

### 5.2 审计日志管理器

```cpp
// ==================== audit_log_manager.h ====================
#pragma once

#include "audit_log.h"
#include <fstream>
#include <queue>
#include <mutex>

namespace aios {

class AuditLogManager {
public:
    AuditLogManager(const std::string& log_dir = "/root/aios/system/audit") 
        : log_dir_(log_dir) {
        // 确保日志目录存在
        std::filesystem::create_directories(log_dir_);
        
        // 启动日志写入线程
        write_thread_ = std::thread(&AuditLogManager::write_worker, this);
    }
    
    ~AuditLogManager() {
        running_ = false;
        write_thread_.join();
    }
    
    // 记录日志
    void log(const AuditLogEntry& entry) {
        std::lock_guard<std::mutex> lock(queue_mutex_);
        log_queue_.push(entry);
    }
    
    // 快捷方法
    void log_error(uint64_t trace_id, pid_t agent_id, 
                   const std::string& error_type, const std::string& details) {
        AuditLogEntry entry;
        entry.timestamp = get_current_timestamp_ms();
        entry.trace_id = trace_id;
        entry.agent_id = agent_id;
        entry.action = AuditAction::EXCEPTION;
        entry.level = AuditLevel::ERROR;
        entry.result_code = -1;
        entry.details = error_type + ": " + details;
        log(entry);
    }
    
    void log_permission_denied(uint64_t trace_id, pid_t agent_id,
                                const std::string& resource) {
        AuditLogEntry entry;
        entry.timestamp = get_current_timestamp_ms();
        entry.trace_id = trace_id;
        entry.agent_id = agent_id;
        entry.action = AuditAction::PERMISSION_DENIED;
        entry.level = AuditLevel::WARNING;
        entry.result_code = 3001;  // PERMISSION_DENIED
        entry.resource = resource;
        log(entry);
    }
    
private:
    void write_worker() {
        while (running_) {
            std::vector<AuditLogEntry> batch;
            
            // 收集一批日志
            {
                std::lock_guard<std::mutex> lock(queue_mutex_);
                while (!log_queue_.empty() && batch.size() < 100) {
                    batch.push_back(log_queue_.front());
                    log_queue_.pop();
                }
            }
            
            // 写入磁盘
            if (!batch.empty()) {
                write_to_disk(batch);
            }
            
            // 休眠
            std::this_thread::sleep_for(std::chrono::milliseconds(100));
        }
    }
    
    void write_to_disk(const std::vector<AuditLogEntry>& entries) {
        std::string filename = log_dir_ + "/audit_" + 
                               std::to_string(get_current_timestamp_ms() / 86400000) + 
                               ".jsonl";
        
        std::ofstream file(filename, std::ios::app);
        for (const auto& entry : entries) {
            file << entry.to_json() << "\n";
        }
    }
    
    std::string log_dir_;
    std::queue<AuditLogEntry> log_queue_;
    std::mutex queue_mutex_;
    std::thread write_thread_;
    bool running_ = true;
};

} // namespace aios
```

## 六、重试机制

### 6.1 重试策略

```python
class RetryStrategy:
    """重试策略"""
    
    def __init__(self, max_retries=3, base_delay=1.0, max_delay=60.0):
        self.max_retries = max_retries
        self.base_delay = base_delay
        self.max_delay = max_delay
    
    def should_retry(self, exception: Exception, attempt: int) -> bool:
        """判断是否应该重试"""
        if attempt >= self.max_retries:
            return False
        
        # 检查异常类型
        retryable_codes = {
            1002,  # LLM_TIMEOUT
            1301,  # HTTP_ERROR
            1302,  # HTTP_TIMEOUT
            2201,  # IPC_ERROR
            2202,  # IPC_TIMEOUT
            3301,  # TIMEOUT
        }
        
        if hasattr(exception, 'code'):
            return exception.code in retryable_codes
        
        return False
    
    def get_delay(self, attempt: int) -> float:
        """计算延迟 (指数退避)"""
        delay = self.base_delay * (2 ** attempt)
        return min(delay, self.max_delay)
```

### 6.2 重试装饰器

```python
def retry_on_error(max_retries=3, base_delay=1.0):
    """重试装饰器"""
    def decorator(func):
        async def wrapper(*args, **kwargs):
            strategy = RetryStrategy(max_retries, base_delay)
            
            for attempt in range(max_retries):
                try:
                    return await func(*args, **kwargs)
                except Exception as e:
                    if not strategy.should_retry(e, attempt):
                        raise
                    
                    delay = strategy.get_delay(attempt)
                    logging.warning(f"Retry {attempt + 1}/{max_retries} "
                                   f"after {delay}s: {e}")
                    await asyncio.sleep(delay)
            
        return wrapper
    return decorator
```
