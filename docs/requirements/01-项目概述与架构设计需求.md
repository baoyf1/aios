## 一、项目背景

### 1.1 项目目标

AIOS（AI Operating System）是一个操作系统级高内聚低耦合的超级架构系统，旨在实现：

```
┌─────────────────────────────────────────────────────────────────────┐
│                         AIOS 项目目标                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. 每个细分领域都有一个Agent                                       │
│     - 文件整理Agent (FileAgent)                                    │
│     - 系统软件调用Agent (SysAgent)                                 │
│     - 邮件Agent (MailAgent)                                        │
│     - 代码Agent (CodeAgent)                                        │
│     - 数据分析Agent (DataAgent)                                    │
│     - 笔记Agent (NoteAgent)                                        │
│     - 搜索Agent (SearchAgent)                                      │
│     - 媒体处理Agent (MediaAgent)                                   │
│     - 网络监控Agent (NetMonitorAgent)                               │
│     - 记忆管理Agent (MemAgent)                                     │
│     - 日志Agent (LogAgent)                                          │
│     - 安全Agent (SecurityAgent)                                     │
│                                                                      │
│  2. 底层Agent统一编排分配任务                                        │
│     - 类似Linux的PID 0 (init进程)                                   │
│     - 中央Agent作为总调度 (Central Agent / PID 0)                 │
│                                                                      │
│  3. 架构设计原则                                                     │
│     - 高内聚低耦合                                                   │
│     - 类似Linux的5层架构                                            │
│     - 统一的信息交付方式（封装/解包）                                │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 二、架构设计理念

### 2.1 Linux 5层架构参考

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 5: Applications    (应用层)    — 用户程序           │
├─────────────────────────────────────────────────────────────┤
│  Layer 4: Libraries/Shell  (库/Shell层) — glibc, API      │
├─────────────────────────────────────────────────────────────┤
│  Layer 3: System Calls     (系统调用层) — 内核接口         │
├─────────────────────────────────────────────────────────────┤
│  Layer 2: Kernel           (内核层)    — 进程/内存/文件系统│
├─────────────────────────────────────────────────────────────┤
│  Layer 1: Hardware         (硬件层)    — CPU/内存/设备     │
└─────────────────────────────────────────────────────────────┘

通信规则：
• 向下：封装 (Encapsulation) — 添加每层的控制信息
• 向上：解包 (Decapsulation) — 剥离控制信息，还原数据
• 跨层禁止直接通信 — 只允许邻层交互（低耦合）
```

### 2.2 AIOS 5层架构详细定义

#### Layer 5: 应用层 (Application Layer)

```
┌─────────────────────────────────────────────────────────────────────┐
│  Layer 5: Application Layer     (应用层)                              │
│  ─────────────────────────────────────────────────────────────────  │
│  • Central Agent (PID 0) — 任务调度 + Agent Registry               │
│    - 职责: 意图解析、任务规划、Agent分发、结果整合                  │
│    - PID: 0 (类似Linux init进程)                                   │
│    - 特性: 唯一性、持久运行、统一调度                               │
│                                                                      │
│  • Domain Agents — 领域专家                                        │
│    - FileAgent: 文件整理/搜索/安全                                  │
│    - SysAgent: 系统状态/进程管理                                   │
│    - NetAgent: 网络请求/DNS/API调用                                │
│    - MailAgent: 邮件收发/搜索                                      │
│    - CodeAgent: 代码读写/搜索/执行                                  │
│    - DataAgent: 数据查询/转换/报表                                  │
│    - NoteAgent: 笔记CRUD/整理/搜索                                  │
│    - SearchAgent: 网页/知识/文件搜索                               │
│    - MediaAgent: 图片/音视频处理                                   │
│    - 扩展: 支持第三方Agent                                        │
│                                                                      │
│  • Function Modules — 功能模块                                     │
│    - file_ops: 文件操作 (list/read/write/move/delete/search)       │
│    - system_ops: 系统操作 (run_cmd/get_env/list_procs)             │
│    - network_ops: 网络操作 (http_get/post/dns)                     │
│    - memory_ops: 记忆操作 (search/write/read)                       │
│    - llm_ops: LLM操作 (chat/embed/stream_chat)                     │
│                                                                      │
│  • LangChain/LangGraph — 工作流编排                                 │
│    - 位置: 内嵌在Domain Agent内部                                   │
│    - 节点类型: LLM决策 / 函数调用 / 条件分支 / 循环                 │
│    - 状态管理: StateGraph                                           │
└─────────────────────────────────────────────────────────────────────┘
```

#### Layer 4: 中间件层 (Middleware Layer)

```
┌─────────────────────────────────────────────────────────────────────┐
│  Layer 4: Middleware Layer      (中间件层)                            │
│  ─────────────────────────────────────────────────────────────────  │
│  • 消息序列化                                                        │
│    - JSON: 开发环境、API、调试                                      │
│    - Protobuf: 生产环境、大量数据                                   │
│                                                                      │
│  • 会话管理 (Session Manager)                                       │
│    - 会话创建/销毁/过期                                             │
│    - 会话状态维护                                                    │
│    - 多会话并发支持                                                  │
│                                                                      │
│  • Python/C++ 绑定                                                   │
│    - PyBridge: C++调用Python                                       │
│    - 数据类型转换                                                    │
│    - 内存管理                                                        │
│                                                                      │
│  注意事项: LangChain/LangGraph已移入Domain Agent内部               │
└─────────────────────────────────────────────────────────────────────┘
```

#### Layer 3: 系统调用/驱动层 (System Call / Driver Layer)

```
┌─────────────────────────────────────────────────────────────────────┐
│  Layer 3: System Call / Driver  (系统调用/驱动层)                      │
│  ─────────────────────────────────────────────────────────────────  │
│  • 驱动模块 (Drivers) — 可插拔                                      │
│    - FS_Driver: 文件系统驱动 (优先级100)                            │
│    - Net_Driver: 网络通信驱动 (优先级90)                            │
│    - LLM_Driver: LLM调用驱动 (优先级80)                            │
│    - Msg_Driver: 消息网关驱动 (优先级70)                           │
│    - Mem_Driver: 记忆存储驱动 (优先级60)                           │
│    - Skill_Driver: Skill加载驱动 (优先级50)                        │
│    - MCP_Driver: MCP协议驱动 (优先级40)                            │
│    - Proc_Driver: 进程管理驱动 (优先级30)                           │
│                                                                      │
│  • 权限守卫 (Permission Guard)                                       │
│    - PathGuard: 路径权限检查                                       │
│    - CapChecker: Capability能力检查                                 │
│    - QuotaGuard: 配额检查 (磁盘/流量/API)                          │
│    - AuditLog: 审计日志                                             │
│                                                                      │
│  • DriverManager — 驱动管理器                                       │
│    - 驱动加载/卸载                                                  │
│    - 驱动注册表                                                     │
│    - 健康检查                                                        │
│    - 请求调度                                                        │
└─────────────────────────────────────────────────────────────────────┘
```

#### Layer 2: 内核层 (Kernel Layer)

```
┌─────────────────────────────────────────────────────────────────────┐
│  Layer 2: Kernel Layer           (内核层)                              │
│  ─────────────────────────────────────────────────────────────────  │
│  • 中央Agent (Agent-0, PID 0角色)                                    │
│    - 进程ID: 0                                                      │
│    - 职责: 进程创建/销毁/调度                                        │
│                                                                      │
│  • O(1)调度器                                                        │
│    - 优先级数组: runqueue[5]                                        │
│    - 位图: bit_map (快速查找非空队列)                                │
│    - 特性: 常数时间复杂度                                           │
│                                                                      │
│  • 内存管理 (per-agent memory system)                                │
│    - 每个Agent独立内存空间                                           │
│    - 内存隔离                                                       │
│    - 内存配额                                                       │
│                                                                      │
│  • 进程管理                                                          │
│    - spawn/kill/wait                                               │
│    - 进程状态机                                                      │
│    - 父子关系                                                        │
│                                                                      │
│  • IPC总线 (进程间通信)                                             │
│    - 消息队列                                                        │
│    - 管道                                                            │
│    - 共享内存                                                        │
│                                                                      │
│  • 安全沙箱                                                          │
│    - 权限控制                                                        │
│    - 审计日志                                                        │
│    - 资源限制                                                        │
└─────────────────────────────────────────────────────────────────────┘
```

#### Layer 1: 硬件层 (Hardware Layer)

```
┌─────────────────────────────────────────────────────────────────────┐
```

## 四、核心设计原则

```
┌─────────────────────────────────────────────────────────────────────┐
│                         核心设计原则                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. 底层不关心上层                                                   │
│     Layer 1-3 不感知 Domain Agent 的业务逻辑                        │
│     通过系统调用和 Driver 接口进行交互                               │
│     好处: 底层可复用，上层灵活变化                                   │
│                                                                      │
│  2. 领域自治                                                        │
│     Domain Agent 独立管理自己的 LangGraph 和 Function Modules       │
│     Central Agent 只负责调度，不管具体实现                           │
│     好处: 单一职责，易于扩展                                        │
│                                                                      │
│  3. 严格分层                                                        │
│     跨层通信必须通过邻层接口                                         │
│     禁止跨层直接调用                                                 │
│     好处: 接口清晰，依赖明确                                         │
│                                                                      │
│  4. 可追溯                                                          │
│     每个请求都有 trace_id                                            │
│     贯穿整个调用链                                                   │
│     好处: 问题定位，性能分析                                         │
│                                                                      │
│  5. 权限最小化                                                      │
│     Agent 只能访问自己工作目录                                       │
│     共享资源需要显式授权                                             │
│     好处: 安全隔离，风险控制                                        │
│                                                                      │
│  6. 事件驱动                                                        │
│     使用 EventChannel + WaitQueue                                   │
│     Agent间通过事件异步通信                                         │
│     好处: 高并发，低阻塞                                            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 五、信息流：封装与解包

### 5.1 完整流程

```
用户输入 ──▶ Layer 5 应用层 (意图分析/任务规划)
    ──▶ Layer 4 中间件 (序列化/会话管理)
    ──▶ Layer 3 系统调用 (权限检查/驱动调度)
    ──▶ Layer 2 内核 (O(1)调度/进程管理)
    ──▶ Layer 1 硬件 (执行: LLM/文件/网络)
    ──▶ 结果逆向逐层返回
```

### 5.2 协议头汇总

| 层级 | 协议头 | 包含关键字段 |
|-----|--------|-------------|
| Layer 1 | HardwareRequest | request_id, operation, raw_result |
| Layer 2 | KernelRequest | request_id, from_pid, to_pid, priority, time_slice |
| Layer 3 | SysCallRequest | request_id, caller_pid, syscall_id, limits, params |
| Layer 4 | MiddlewareMessage | trace_id, agent_id, timestamp, session_id, payload |
| Layer 5 | AppRequest | raw_input, Intent, Tasks |

## 六、功能需求

### 6.1 核心功能 (P0)

| 功能 | 描述 | 实现要求 |
|-----|------|---------|
| 5层架构实现 | 按照Linux分层思想实现5层架构 | 每层职责清晰，接口明确 |
| 中央Agent | PID 0角色，统一调度所有Agent | 意图解析、任务规划、结果整合 |
| Domain Agent | 领域专家Agent，可扩展 | 每个Agent独立LangGraph工作流 |
| 层间通信 | 严格的封装/解包流程 | trace_id贯穿整个调用链 |
| O(1)调度器 | 优先级数组 + 位图调度 | 常数时间复杂度 |
| IPC总线 | 进程间通信机制 | 支持同步/异步消息 |
| 驱动架构 | 可插拔驱动模块 | DriverManager统一管理 |

### 6.2 重要功能 (P1)

| 功能 | 描述 | 实现要求 |
|-----|------|---------|
| 权限控制 | PathGuard + CapChecker + QuotaGuard | 审计日志完整 |
| 多Agent协作 | Context Chain + trace_id追溯 | 支持父子/同级调用 |
| 事件驱动 | EventChannel + WaitQueue | GlobalEventLoop统一管理 |
| 超级记忆系统 | 向量数据库 + 渐进披露 | Memory GC自动整理 |
| 安全沙箱 | Sandbox隔离 + 配额控制 | commit时审计 |

### 6.3 优化功能 (P2)

| 功能 | 描述 | 实现要求 |
|-----|------|---------|
| 第三方Agent开发 | Agent开发模板 | 支持动态加载 |
| 插件系统 | Plugin机制 | VSCode/IDE集成 |
| Web界面 | 用户界面 | 可选实现 |

## 七、非功能需求

### 7.1 性能

| 指标 | 要求 | 说明 |
|-----|------|------|
| 调度延迟 | < 1ms | O(1)调度器保证 |
| Agent响应时间 | < 5s | 典型任务 |
| 并发Agent数 | >= 100 | 支持多任务 |
| 内存占用 | < 2GB | 基础运行 |

### 7.2 可用性

| 指标 | 要求 | 说明 |
|-----|------|------|
| 运行时间 | 7x24 | 持续运行 |
| 热加载 | 支持 | Agent/驱动热更新 |
| 故障恢复 | 自动 | 进程崩溃恢复 |

### 7.3 安全

| 指标 | 要求 | 说明 |
|-----|------|------|
| 沙箱隔离 | 必须 | Agent间隔离 |
| 审计日志 | 完整 | 所有敏感操作 |
| 权限控制 | 最小权限 | 默认拒绝 |

### 7.4 可扩展

| 指标 | 要求 | 说明 |
|-----|------|------|
| 第三方Agent | 支持 | 开发模板 |
| 新驱动 | 支持 | 可插拔架构 |
| 新消息渠道 | 支持 | 消息网关抽象 |
